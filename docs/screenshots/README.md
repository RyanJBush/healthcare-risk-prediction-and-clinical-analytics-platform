## Capture guidance

- Use a clean browser window at roughly **1440×900** so screenshots fit on a portfolio page without scrolling.
- Prefer **light mode** unless the design intentionally targets dark mode — light-mode shots reproduce better on resumes and slide decks.
- Crop to the relevant pane; avoid capturing tab bars, bookmarks, or OS chrome.
- **Synthetic identifiers only.** Everything in the seeded cohort is generated; do not paste real names, MRNs, or any PHI into the UI before capturing.
- Save as **PNG**, named exactly as in the table above so README links resolve. Keep individual files under ~500 KB if possible (use `pngquant` or similar).
- For the fairness-report and API screenshots, terminal captures are fine — use a monospace font and a dark terminal background for contrast.
