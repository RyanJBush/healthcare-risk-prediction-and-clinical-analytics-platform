# Cerberus Architecture

> **Not for clinical use.** All data is synthetic. Cerberus is a portfolio project, not a validated medical product.

## High-level shape

Cerberus is a three-tier local application:

```
React (Vite) UI  ──►  FastAPI (~49 endpoints)  ──►  PostgreSQL / SQLite
                            │
                            ├─► Tiered ML scoring pipeline (heuristic → scikit-learn)
                            ├─► SHAP-style explanations
                            ├─► Drift monitoring scaffold
                            └─► Model registry + training-run history
```

A Mermaid version lives in the [README](../README.md#%EF%B8%8F-architecture).

## Backend

- **Framework:** FastAPI (`backend/app/main.py`) with a lifespan handler that creates tables and seeds demo users on startup.
- **Persistence:** SQLAlchemy 2.x against PostgreSQL in Docker Compose, or SQLite for tests. Models in `backend/app/models.py` cover `Patient`, `Observation`, `Prediction`, `Explanation`, `TrainingRun`, `EvaluationRun`, `ModelCard`, `ModelConfigChange`, `AuditLog`, `PatientAccessGrant`, `BatchScoringJob`, `ReviewNote`, and `User`.
- **Auth:** JWT bearer tokens issued at `POST /auth/login`. RBAC enforced at the route level via `security.require_roles(...)`. Four seeded roles: `clinician`, `admin`, `analyst`, `viewer`.
- **Routers:** Endpoints are defined directly on the main `FastAPI` app — roughly 49 routes spanning patients, observations, predictions, explanations, triage, model registry, training, evaluation, monitoring (drift + prediction log), audit, access grants, cohorts, reports, and summaries.

## ML scoring pipeline

- **Targets (3):** `readmission` (30-day), `deterioration`, `adverse_event`. Each has its own weight vector, intercept, and risk thresholds in `TARGET_CONFIGS` (`backend/app/ml.py`).
- **Base features (6):** `age`, `bmi`, `blood_pressure`, `cholesterol`, `glucose`, `smoker` (see `FEATURE_NAMES`). Derived signals (composite indices, protective factors) are computed in `predict_tiered`.
- **Tiered inference:** `predict_tiered` runs a fast weighted logistic-style score first. Only low-confidence cases are escalated to the full scikit-learn path.
- **Full ML path:** `backend/app/services/risk_models.py` and `backend/app/services/training.py` train `LogisticRegression` and `RandomForestClassifier` pipelines (scikit-learn) with `StandardScaler` preprocessing. Training runs are persisted as `TrainingRun` rows.
- **Explainability:** SHAP (`shap==0.46.0`) feeds top-factor attributions; the API also returns reason codes and a plain-language rationale per prediction.
- **Evaluation:** `backend/app/services/evaluation.py` computes ROC-AUC, PR-AUC, Brier score, F1, precision, recall, and a confusion matrix.

## Trust & responsible-ML scaffolding

- **Model registry:** `GET /api/model-registry` lists registered models; `POST /api/training/runs` trains and registers a new version. Every prediction is traceable back to a model version.
- **Drift detection:** `GET /api/monitoring/drift` compares request-time prediction distributions against the training baseline. Values are illustrative on synthetic traffic.
- **Fairness slicing:** `scripts/fairness_eval.py` reports per-slice positive-rate, mean risk score, risk-category distribution, and disparity metrics on a synthetic two-group cohort. Offline only.
- **Audit log:** `AuditLog` rows persist key actions (scoring, training, threshold change, patient access). Exposed at `GET /api/audit/logs`.
- **Model card:** Intended use, metrics, biases, and limitations live in [`MODEL_CARD.md`](../MODEL_CARD.md).

## Frontend

- React 19 + Vite + Tailwind CSS + Recharts, in **JavaScript** (`.jsx`, not TypeScript).
- Pages: `LoginPage`, `DashboardPage`, `PatientsPage`, `PatientDetailPage`, `TriagePage`, `RiskAnalysisPage` (`frontend/src/pages/`).
- API client: `frontend/src/api.js` — stores the JWT and adds the bearer header on each request.

## Data

- All data is **synthetic**. Generated by `backend/app/services/seed_loader.py` and surfaced via `POST /api/demo/load-seed`.
- No real names, MRNs, or any PHI. Masked synthetic identifiers only.
- Persisted in PostgreSQL when running under Docker Compose; SQLite is used for local tests (`DATABASE_URL=sqlite+pysqlite:///./nova_test.db`).

## Infra & quality

- **Docker Compose** (`docker-compose.yml`) brings up `db` (Postgres 16), `backend` (FastAPI), and `frontend` (Vite dev server).
- **Makefile** standardizes `make doctor / setup / dev / lint / test / demo-bootstrap`.
- **CI** (`.github/workflows/ci.yml`) runs ruff (lint), mypy (type-check, non-blocking), pytest, frontend lint, and a Vite production build on every push and PR to `main`.
- **Tests:** 11 pytest modules in `backend/tests/` covering API routes, ML behavior, training, evaluation, security, seeding, model registry, summaries, risk models, and the CLI scripts.

## Non-goals

- Cerberus is **not** clinically validated, **not** a medical device, **not** HIPAA-scoped, and **not** deployed. It is a portfolio artifact meant to demonstrate responsible-ML patterns on synthetic data.
